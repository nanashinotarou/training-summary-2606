## 1. Antigravityの担当フェーズ（1日の流れ Step 1）

Hiroyaから「今日のコース情報」を受け取ったら、以下を順番に実行する。
ClaudeCode・Cursorへの引き渡しまでが担当範囲。

### Step 1-1：コース情報の整形 → Course_Log.md に追記

Hiroyaから渡されるコース情報（Day番号・日付・テーマ・YouTube URL等）を以下のフォーマットに整形して `Course_Log.md` に追記する。

```markdown
## Day XX — YYYY/MM/DD (曜)
**テーマ:** [テーマ名]
**YouTube:** [URL]
**備考:** [Hiroyaからの補足があれば記載、なければ省略]
```

- 余分な改行・全角スペース・箇条書きの揺れなどを統一すること
- 既存エントリは絶対に編集しない。**末尾に追記のみ**

### Step 1-2：YouTube文字起こし → Today_Research.md に書き出し

対象動画を文字起こしし、以下の構成で `Today_Research.md` を**上書き**する。

```markdown
# Today_Research.md
> Day XX「[テーマ名]」リサーチ結果 — YYYY/MM/DD

## 動画情報
- タイトル：
- URL：
- 時間：

## 要点まとめ（ClaudeCode・Cursor向け）
### [セクション名]
- 要点1
- 要点2
...

## 教材に使えそうなキーワード・フレーズ
- 

## 注意点・補足
- 
```

- 文字起こし原文は保存しない（VTTファイルは処理後に削除する）
- 要点まとめは**ClaudeCodeが実装計画を立てやすい粒度**で書く
- 1動画あたり300〜600字程度を目安にまとめる

### Step 1-3：Today_Research.md 品質チェック（引き渡し前に必ず確認）

`Today_Research.md` を書き終えたら、ClaudeCodeに引き渡す前に以下を自己チェックすること。

| # | チェック項目 | 基準 |
|---|---|---|
| 1 | Day番号・日付・テーマが正確に記載されているか | Course_Log.md の当日エントリと一致 |
| 2 | YouTube URLが記載されているか | 文字起こし元と同じURL |
| 3 | 動画タイトルと尺（分）が記載されているか | 省略禁止 |
| 4 | 要点まとめが **3セクション以上** あるか | セクション数＝ページの章数の目安になる |
| 5 | 各セクションに **箇条書き2項目以上** あるか | 1行だけのセクションはNG |
| 6 | 「教材に使えそうなキーワード・フレーズ」に **3語以上** あるか | ClaudeCodeがページタイトル・見出しを作る素材になる |
| 7 | 「注意点・補足」が記載されているか | 特になければ「特になし」と明記（空欄禁止） |
| 8 | **リサーチタイトル（テーマ名）と実際の動画内容が一致しているか** | カリキュラム上のDay名と動画内容がズレる場合は、実際の動画内容に合わせたテーマ名に修正して記載する（Day02で「企業が運用をする目的」というDay名に対し実際はCanva動画だった問題） |

全項目OKならClaudeCodeへ引き渡す。不足があれば補完してから引き渡すこと。

### Step 1-4：NotebookLM プロンプトの生成

`Today_Research.md` の品質チェックが完了したら、**その日の授業内容に特化した NotebookLM プロンプト**を生成してHiroyaに渡す。

#### プロンプトの構成（固定部分 ＋ 変動部分）

以下のテンプレートの【変動部分】を、その日のリサーチ内容に基づいて埋めること。

```
Instagram運用を学ぶ社会人初心者向けの研修スライドです。
プレゼンターのスライド形式で、一枚ごとに重要ポイントを1〜2つに絞ってください。
明るく親しみやすいビジュアルで、シンプルで視覚的に分かりやすくしてください。

【今日の授業内容】
テーマ：[今日のDay番号とテーマ名]
実習内容：[今日の実習・ワーク内容を1文で]
重点的に図解してほしいポイント：
- [視覚化すべき操作手順・概念 1]
- [視覚化すべき操作手順・概念 2]
- [視覚化すべき操作手順・概念 3]
```

#### 変動部分の埋め方ルール

| フィールド | 何を書くか |
|---|---|
| テーマ | `Day XX「[テーマ名]」` の形式で記載 |
| 実習内容 | 受講者が「手を動かしてやること」を1文で（例：「Canvaでリールを1本作る」） |
| 図解ポイント① | 動画内で最も多く言及された操作・手順 |
| 図解ポイント② | 受講者が一番つまずきやすそうな概念・ステップ |
| 図解ポイント③ | まとめ・比較・全体像として見せると効果的な内容 |

#### 引き渡し報告テンプレート

```
リサーチ完了しました。

【NotebookLM プロンプト（コピペしてください）】
---
（上記で生成したプロンプトをここに貼る）
---

【NotebookLM 操作手順】
1. NotebookLM（https://notebooklm.google.com）で新規ノートブックを作成
2. 今日の全動画URLをソースとして追加
3. 「ノートブックガイド」→「スライド」→「プレゼンターのスライド」を選択
4. 言語：日本語 / 長さ：デフォルト
5. 「説明」欄に上記プロンプトをコピペ → 生成
6. 全スライドを assets/ に 1.png, 2.png... の連番で保存（選別不要）
7. 準備ができたら ClaudeCode に引き渡し
```

- 画像がない場合でも ClaudeCode はテキストのみで実装できるため、画像は任意扱い
- スライドの選別（どれをHTMLに使うか）は ClaudeCode が行う
- **ClaudeCode が画像を見た後にプロンプトの精度評価を行い、改善点を Today_Plan.md にフィードバックする**

### Step 1-5 ← 旧Step1-4（参照用） NotebookLM スライド生成の補足手順

**① ノートブック作成**
- NotebookLM（https://notebooklm.google.com）で「新しいノートブック」を作成
- ソース追加 → 「YouTube」を選択 → 当日の全動画URLを1本ずつ追加（複数OK）

**② スライド生成**
- 右側の「ノートブックガイド」パネルを開く
- 「スライド」をクリック →「スライド資料をカスタマイズ」ダイアログが開く

**③ カスタマイズダイアログの設定**

| 項目 | 設定値 |
|---|---|
| タイプ | **プレゼンターのスライド**（ビジュアル重視でサイト映え◎） |
| 言語 | 日本語 |
| 長さ | デフォルト |

「作成するスライドについて説明してください」欄に以下をコピペ：

```
Instagram運用を学ぶ社会人初心者向けの研修スライドです。
一枚ごとに重要ポイントを1〜2つに絞り、シンプルで視覚的に分かりやすくしてください。
専門用語には短い説明を添え、具体的な操作例や実践的なTipsを入れてください。
```

→「生成」をクリック

**④ 保存**
- 生成されたスライドを1枚ずつドラッグ＆ドロップ or 右クリック保存
- ファイル名は自動付与（`Master_..._Slide_N.png`）→ ClaudeCode がリネームするのでそのままでOK
- `assets/` フォルダに全枚数保存（選別不要）

## 2. 一時ファイルの運用ルール

- **`Today_Research.md`**: 上書き更新。新規ファイルを作らない
- **VTTファイル**: 処理完了後に必ず削除
- **スクラッチスクリプト**: `scratch/` フォルダに格納し、不要になったら削除
- **引き継ぎメモ等の別ファイル**: 作らない。`Today_Research.md` に集約する
